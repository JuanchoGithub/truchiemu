Development

TruchiEmu is built with a modern, modular architecture combining SwiftUI for the interface, Metal for shader effects, and the libretro API for emulation core integration.

Architecture Overview

TruchiEmu's architecture is organized into several key layers:

App Layer (SwiftUI)

  • App entrypoint: TruchiEmuApp.swift and ContentView.swift
  • Views: SwiftUI views in TruchiEmu/Views/ and TruchiEmu/Features/
  • Data layer: SwiftData models in TruchiEmu/Core/Models/

Emulation Engine (Objective-C++ / C)

  • Libretro bridge: TruchiEmu/Core/Engine/ — mixed ObjC++/C with a Swift bridging header
  • Swift-ObjC bridge: LibretroBridge.mm / LibretroBridgeSwift.swift

Graphics (Metal)

  • Shader files: TruchiEmu/Core/Shaders/
  • Runtime shaders compiled and executed on the GPU via Metal

Services

  • Save management: SaveDirectoryManager and SaveMigrationService
  • Library management: ROM scanning, metadata fetching, artwork management

Building from Source

Prerequisites

  • macOS 14.0 (Sonoma) or later
  • Xcode 15+ with Swift 5.9+
  • XcodeGen (install via brew install xcodegen)
  • Xcode Command Line Tools (xcode-select --install)

Build Steps

  1. Clone the repository: git clone https://github.com/JuanchoGithub/truchiemu.git
  2. Navigate to the project: cd truchiemu
  3. Generate Xcode project: xcodegen generate
  4. Open TruchiEmu.xcodeproj in Xcode
  5. Build with Cmd+B or: xcodebuild -scheme TruchiEmu -configuration Debug build

Important Notes

  • Do not edit the .xcodeproj directly — edit project.yml and run xcodegen generate
  • When adding source files, add paths under the appropriate target's sources in project.yml
  • C++ standard: gnu17/gnu++17 (not LLVM default)
  • Bridging header: TruchiEmu-Bridging-Header.h exposes ObjC++ symbols to Swift

Running Tests

Tests live in TruchiEmuTests/ and can be run with: xcodebuild test -scheme TruchiEmuTests -destination 'platform=macOS'
