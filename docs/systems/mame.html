System Overview

MAME (Multiple Arcade Machine Emulator) is not a single system but a preservation project covering thousands of arcade systems. Instead of emulating a specific hardware platform, MAME emulates the original arcade hardware, ROMs, and CPU architectures of countless arcade machines spanning from 1975 to the modern era.

ROM Format Support

ROM Set Types:

  • Split: Common files shared across clones
  • Non-Merged: All files in single archive
  • Merged: Parent and clones combined

Compression Formats:

  • .zip - Standard compression
  • .7z - Advanced compressed archives
  • .chd - Compressed hard disk for media (CDs, HDD)
  • .bin - Binary dumps

BIOS Requirements:

Many systems require specific BIOS files in the system directory. Examples include:

  • neogeo.zip for Neo Geo games
  • cps2.zip for various Capcom games
  • Various BIOS files for different hardware sets

Best Shader Recommendations

Arcade games spanned multiple decades and technologies. Choose shaders based on original display:

Early Raster Games (1975-1990):

  • CRT-Geom: Standard CRT simulation
  • CRT-PI: Lightweight with scanlines

Late 80s/Early 90s:

  • CRT-Royale-Aperture: High-end CRT
  • MDAPT: Fixes dithering artifacts

3D Games (Mid-90s+):

  • FXAA/SMAA: Edge smoothing
  • ScaleFX: Upscaling for low-res textures

Performance Optimization

MAME's accuracy comes at a significant CPU cost:

System Requirements:

  • z80-based games: Low CPU requirement
  • 68k-based games: Moderate CPU
  • 3D hardware: High CPU, GPU accelerated rendering preferred
  • Modern MAME: Heaviest, requires current hardware

Speed Optimization:

  • Frame skipping: 0 or 1 maximum
  • Audio latency: 128-256ms
  • DRC (Dynamic Recompiler): Enable when available
  • HLE vs LLE: Use HLE for performance when available

Known Issues and Workarounds

ROM Set Mismatch:

Issue: Games fail to load with "missing files" errors.

Workaround: Ensure ROM version matches MAME core version exactly. Use ROM managers like ClrMamePro to verify sets.

CHD Format Nuances:

Issue: CHD files incorrectly compressed or structured.

Workaround: Use chdman tool from MAME distribution to create CHDs. Store in path matching MAME's expected structure.

Input Mapping Complexity:

Issue: Complex control schemes with multiple players, coin inputs, etc.

Workaround: Map coin buttons to shoulder triggers, service buttons to F-keys. Use core input remapping per-game.

Vector Games Performance:

Issue: Vector games (Asteroids, Battlezone) run slowly at high resolutions.

Workaround: Lower internal resolution, enable "Antialias" for vector beam simulation, use shaders for phosphor glow.

System Selection: Not all games run well in all MAME cores. Older cores may run some games better but lack compatibility with newer ROM dumps.

Save State Support:

Issue: Some drivers lack save state support, causing crashes.

Workaround: Verify save state support in core info. Use nvram saves for games with modified NVRAM storage.
