System Requirements

🖥️ macOS Compatibility

Minimum: macOS 14.0 (Sonoma) or later with Swift 5.9+

Recommended: macOS 15.0 (Sequoia) or later for optimal performance

Hardware Requirements

Component Minimum Recommended Notes
Processor Intel Core i5 or Apple M1 Apple M1 Pro/Max/Ultra or Intel Core i7+ ARM64 native on Apple Silicon
RAM 8 GB 16 GB or more More RAM enables larger game libraries
Storage 2 GB free space 10 GB+ free space For game ROMs and save states
Graphics Intel Iris Plus or Apple M1 GPU Dedicated GPU or Apple M1 Pro+ Required for shaders and upscaling

Software Dependencies

  • Xcode Command Line Tools: Required for building from source
  • Metal-compatible GPU: For shader rendering and hardware acceleration
  • Network access: For downloading cores, thumbnails, and updates
  • ROM files: Legally obtained game backups (we do not provide ROMs)

⚠️ Important: TruchiEmu requires macOS 14.0 or later. The application uses modern SwiftUI features and Metal shaders that are not compatible with older macOS versions. Ensure your system is up to date before proceeding with installation.

Installation Methods

Method 1: Download Pre-built Binary (Recommended)

The easiest way to get started with TruchiEmu is to download the latest pre-built release from our GitHub releases page.

Step-by-Step Installation:

  1. Visit our GitHub releases page
  2. Download the latest TruchiEmu-x.x.x.dmg file
  3. Open the DMG file and drag TruchiEmu to your Applications folder
  4. Launch TruchiEmu from Applications or Launchpad
  5. If you see a security warning, right-click the app and select "Open"

💡 Tip: The DMG installation is cryptographically signed and notarized by Apple, ensuring it's safe to run on your Mac without disabling Gatekeeper security features.

Method 2: Install via Homebrew

For users who prefer package management, TruchiEmu is available through Homebrew.

brew tap juancho-github/truchiemu
brew install truchiemu

To upgrade to the latest version:

brew update
brew upgrade truchiemu

Advantages of Homebrew Installation:

  • Automatic updates through brew upgrade
  • Easy uninstall with brew uninstall truchiemu
  • Command-line integration
  • Dependencies handled automatically

Method 3: Build from Source

For developers and advanced users who want the latest features or want to contribute to the project.

Prerequisites:

xcode-select --install  # Install Xcode Command Line Tools
brew install cmake xcodegen  # Install build dependencies

Build Steps:

git clone https://github.com/JuanchoGithub/truchiemu.git
cd truchiemu
xcodegen generate  # Generate Xcode project
open TruchiEmu.xcodeproj  # Open in Xcode
# Build in Xcode (⌘+B) or run:
xcodebuild -scheme TruchiEmu -configuration Debug build

⚠️ Note: Building from source requires approximately 10 GB of free disk space and may take 15-30 minutes depending on your system.

First Launch Setup

When you launch TruchiEmu for the first time, you'll be guided through an initial setup wizard to configure your emulation environment.

The main settings interface where you configure global preferences

Setup Wizard Steps

Step 1: Welcome & Privacy

Review and accept the privacy policy and data collection settings. TruchiEmu collects anonymous usage statistics to improve the app but never collects personal information or ROM data.

Step 2: Library Location

Choose where to store your game library and save files. The default location is ~/Documents/TruchiEmu/. This directory will contain:

  • ROMs/ - Your game files
  • Saves/ - Save states and battery saves
  • Cores/ - Downloaded emulation cores
  • Thumbnails/ - Game artwork and screenshots

Step 3: Controller Setup

Connect and configure your preferred input method. TruchiEmu supports:

  • Keyboard input (default)
  • Popular controllers (Xbox, PlayStation, Nintendo Switch)
  • Generic USB gamepads
  • Custom button mapping per system

Step 4: Core Download

Download essential emulation cores. We recommend starting with popular systems like NES, SNES, Game Boy, and Genesis. Additional cores can be downloaded later from the Core Management menu.

The intuitive controller mapping interface supports per-system configuration

💡 Pro Tip: Take time to properly configure your controller during first setup. A well-mapped controller significantly improves the gaming experience, especially for complex systems like N64 or PlayStation.

Adding Your First Game

Once you've completed the initial setup, it's time to add games to your library. TruchiEmu supports drag-and-drop import and automatic game detection.

The library grid view automatically organizes your games with cover art

Import Methods

Method 1: Drag and Drop

  1. Open the TruchiEmu library window
  2. Locate your legally obtained ROM files in Finder
  3. Drag and drop ROM files directly into the TruchiEmu window
  4. The games will automatically be scanned, identified, and added to your library
  5. Cover art and metadata will be downloaded automatically

Method 2: Manual Import

  1. Click the "+" button in the library toolbar
  2. Select "Import ROMs..." from the menu
  3. Browse to your ROM files location
  4. Select one or multiple ROM files
  5. Click "Open" to import

Method 3: Watch Folder

  1. Open Preferences (⌘+,)
  2. Navigate to the Library tab
  3. Enable "Watch for new ROMs in library folder"
  4. Any ROMs added to your library folder will be automatically imported

Supported File Formats

📦 Archives

  • .zip - Compressed ROM archives
  • .7z - 7-Zip compressed files
  • .rar - RAR compressed files

🎮 Raw ROMs

  • .nes - Nintendo Entertainment System
  • .sfc, .smc - Super Nintendo
  • .gba - Game Boy Advance
  • .gbc, .gb - Game Boy
  • .md, .smd - Sega Genesis/Mega Drive

💿 Disc Images

  • .iso - CD/DVD images
  • .cue + .bin - Multi-track images
  • .chd - Compressed Hunks of Data

⚠️ Legal Notice: TruchiEmu does not provide or distribute copyrighted ROM files. You must legally obtain game backups from cartridges or discs you own. We do not condone piracy.

Controller Support

TruchiEmu automatically detects most popular controllers when connected via USB or Bluetooth:

  • Xbox Controllers: Perfect support for Xbox One, Xbox Series X|S controllers
  • PlayStation Controllers: DualShock 4 and DualSense (PS5) supported
  • Nintendo Switch: Joy-Cons and Pro Controller (with proper mapping)
  • Generic USB: Most DirectInput and XInput controllers work out of the box

💡 Controller Tip: For the best experience, we recommend using a modern controller like the Xbox Series X controller or PlayStation DualSense. These offer excellent D-pads, responsive buttons, and wireless connectivity that works seamlessly with macOS.

Per-System Customization

Each emulator core can have its own control mapping:

  1. Open a game for the system you want to configure
  2. Access the Quick Menu (F1 or Escape)
  3. Navigate to Controls
  4. Adjust button mappings as needed
  5. Save as Core Remap File for persistent settings

Save States & Memory Management

Save states allow you to save and load your exact position in a game at any moment, while battery saves work like the original hardware.

The visual save state browser shows thumbnails and timestamps

Save States vs. Battery Saves

Save States

  • Save exact emulator state
  • Instant save and load
  • 8 slots per game
  • Screenshot preview
  • Cross-platform compatible

Battery Saves

  • Save like original hardware
  • Compatible with real hardware
  • Automatic saving
  • No preview images
  • Used by save points in-game

Save State Features

  • 8 Save Slots: Each game has 8 save state slots (F1-F8)
  • Visual Previews: Each save state shows a screenshot thumbnail
  • Timestamps: See exactly when each save was created
  • Automatic Backups: Save states are backed up to prevent loss
  • Compression: Save states are compressed to save disk space
  • Portability: Move save states between computers easily

How to Use Save States

Action Keyboard Menu Path Description
Save to Slot 1 F1 Quick Menu → Save State → Slot 1 Instant save to primary slot
Load from Slot 1 Alt+F1 Quick Menu → Load State → Slot 1 Restore from primary slot
Quick Save Tab System → Quick Save Save to last used slot
Quick Load R System → Quick Load Load from last used slot
Save State Browser F2 Quick Menu → Save State Browser Visual save state manager

💡 Save State Best Practices:

  • Use Slot 1 (F1) for critical progress points
  • Use higher slots (F2-F8) for experimental or practice saves
  • Create manual save states before difficult sections
  • Don't rely solely on save states - use battery saves too
  • Backup your Saves/ folder periodically
