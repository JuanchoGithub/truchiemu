Library Management

TruchiEmu's intelligent library management system automatically organizes your game collection with metadata scraping, artwork downloading, and advanced filtering capabilities. Transform your ROM collection into a beautiful, searchable library with rich game information and box art.

Service Overview

The Library Management service provides comprehensive organization and discovery tools for your game collection. Using machine learning-based metadata matching and integration with multiple game databases, TruchiEmu automatically identifies games, downloads high-quality artwork, and creates an immersive browsing experience that rivals modern game launchers.

Library data flow from ROM scanning to metadata enrichment and UI display

Core Features

  • Automatic Game Detection: AI-powered ROM identification using multiple database sources
  • Metadata Scraping: Pulls game info, descriptions, release dates, and developer information
  • Artwork Management: Automatic download of box art, screenshots, and fan art
  • Smart Collections: Dynamic grouping by system, genre, year, or custom rules
  • Import/Export: Bulk library management with CSV and JSON support
  • Duplicate Detection: Intelligent duplicate finder with region/version differentiation

Technical Architecture

The library system comprises several integrated components:

  • ROM Scanner: Recursive directory scanning with file hash calculation and header extraction
  • Metadata Engine: Multi-source data aggregation from LaunchBox, IGDB, and custom databases
  • Artwork Pipeline: Parallel downloading with format optimization and caching
  • Duplicate Analyzer: Fuzzy matching algorithm to identify true duplicates vs. regional variants
  • SwiftData Backend: Persistent storage with fast search and filtering capabilities
  • Index Service: Full-text search across titles, descriptions, and metadata tags

Step-by-Step Configuration Guide

Initial Library Setup

  1. Add ROM Directories

    Navigate to Settings → Library → ROM Directories and click "Add Directory". Select folders containing your game ROMs. TruchiEmu supports recursive scanning of subdirectories.

  2. Initial Scan Configuration

    Configure scan options: file extensions to include (nes, smc, gb, gba, etc.), exclude patterns for beta/prototype versions, and maximum file size limits for known bad dumps.

  3. Enable Metadata Sources

    In Settings → Library → Metadata, select your preferred metadata providers. LaunchBox is recommended for accuracy, with IGDB as fallback for newer titles.

  4. Start Library Scan

    Click "Scan for New ROMs" in the main Library view. The scanner will calculate hashes, identify games, and fetch metadata. Progress is shown in real-time with estimated completion time.

  5. Review Scan Results

    After scanning, review the "Unidentified Games" section. Manually match unrecognized ROMs by selecting the correct game from search results or creating custom entries.

Advanced Library Organization

  1. Create Smart Collections

    Navigate to Library → Smart Collections → New Collection. Define rules based on system, genre, release year, playtime, or achievement completion status.

  2. Configure Artwork Preferences

    In Settings → Library → Artwork, set your preferences for box art regions (USA, Europe, Japan), artwork quality (Standard, High, Ultra), and download parallelization settings for faster fetching.

  3. Set Up Automatic Artwork Download

    Enable "Auto-download Missing Artwork" to automatically fetch artwork for new games added to the library. Configure bandwidth limits to prevent network saturation during downloads.

  4. Configure Duplicate Detection

    Adjust duplicate detection sensitivity in Settings → Library → Advanced. Set thresholds for considering games duplicates based on title similarity and file content matching.

  5. Create Custom Metadata Fields

    Add custom tags and metadata fields for personal organization. In Settings → Library → Custom Fields, define fields like "Condition", "Purchase Date", or "Rating".

Import and Export Workflows

  1. Export Library Data

    Go to File → Export → Library Data and choose format (JSON, CSV, or XML). Select which fields to include and whether to embed artwork data.

  2. Import from Other Launchers

    Use File → Import to bring libraries from other applications. Supported formats include LaunchBox XML, RetroArch playlists, and generic ROM lists.

  3. Batch Metadata Refresh

    Select multiple games and choose Refresh Metadata from the context menu. This updates all metadata fields and artwork for the selected games.

  4. Duplicate Management

    Run Tools → Find Duplicates to identify duplicate games. Choose to hide, merge, or delete duplicates with intelligent suggestions based on region and version quality.

Configuration Examples

Example 1: Complete Library Scan Configuration

// Library Scanner Configuration
scanning {
    directories = [
        "/Users/name/Games/NES",
        "/Users/name/Games/SNES",
        "/Users/name/Games/Genesis"
    ]
    
    file_extensions = ["nes", "smc", "sfc", "gen", "smd", "bin"]
    
    exclude_patterns = [
        "*beta*",
        "*proto*",
        "*(*hack*)*",
        "*[b]*"           // Bad dumps
    ]
    
    max_file_size = 50_000_000      // 50MB limit
    recursive_scan = true
    follow_symlinks = false
    
    checksum_algorithm = "sha256"
    streaming_checksum = true        // Memory-efficient for large collections
}

Example 2: Multi-Source Metadata Configuration

// Metadata Sources Configuration
metadata {
    primary_source = "launchbox"
    fallback_sources = ["igdb", "custom_db"]
    
    // LaunchBox Preferences
    launchbox {
        api_key = "YourLaunchBoxAPIKey"
        region_preference = ["usa", "world", "europe", "japan"]
        prioritize_official = true
    }
    
    // IGDB Configuration
    igdb {
        client_id = "YourClientID"
        client_secret = "YourClientSecret"
        max_daily_requests = 5000
    }
    
    // Field Priority
    field_priority {
        description = ["launchbox", "igdb"]
        genres = ["igdb", "launchbox"]
        release_date = ["launchbox"]
        developer = ["igdb"]
    }
    
    // Update Behavior
    auto_refresh_age = 30             // Days before metadata refresh
    overwrite_existing = false        // Only fill missing fields
}

Example 3: Artwork Download Pipeline

// Artwork Download Configuration
artwork {
    // Download Settings
    parallel_downloads = 5
    bandwidth_limit = 10_000_000      // 10MB/s (0 for unlimited)
    retry_attempts = 3
    timeout_seconds = 30
    
    // Artwork Preferences
    quality = "high"                  // Options: low, standard, high, ultra
    region_preference = ["usa", "europe", "japan"]
    
    // Cover Images
    cover {
        enabled = true
        dimensions = "800x1000"         // Target dimensions
        format = "jpg"                  // Options: jpg, png, webp
        include_3D_boxes = true
    }
    
    // Screenshots
    screenshot {
        enabled = true
        count = 3                       // Max screenshots per game
        dimensions = "1024x768"
        format = "png"
    }
    
    // Fan Art
    fanart {
        enabled = false                 // Disable to save bandwidth
        count = 2
        dimensions = "1920x1080"
    }
    
    // Cache Management
    cache_location = "~/Documents/TruchiEmu/Cache"
    max_cache_size = 5_000_000_000    // 5GB cache limit
    auto_cleanup = true               // Remove unused artwork
}

Troubleshooting

Games Not Identified

Symptom: ROMs appear as "Unknown Game" in library.

Solution: Verify ROM file integrity by checking checksum. Ensure file extensions are in the allowed list. Try manual lookup via right-click → Identify Game.

Artwork Download Failures

Symptom: Box art and screenshots not downloading.

Solution: Check internet connection and firewall settings. Verify API credentials for metadata sources in settings. Try reducing parallel downloads to avoid rate limiting.

Duplicate Games in Library

Symptom: Same game appearing multiple times in library.

Solution: Run duplicate finder with adjusted sensitivity. Check for different ROM versions (USA vs Europe) which may be legitimate. Use smart collections to filter by region.

Scanning Performance Issues

Symptom: Library scan taking extremely long or freezing.

Solution: Disable streaming checksum for large files. Reduce parallel operations in advanced settings. Scan directories in smaller batches instead of entire collection at once.

Metadata Missing or Incorrect

Symptom: Game information wrong or incomplete.

Solution: Refresh metadata from different source using right-click menu. Add missing information manually using Edit Game dialog. Report incorrect data to metadata providers.

Smart Collection Not Updating

Symptom: Smart collections showing outdated game lists.

Solution: Smart collections update automatically when games are played. Manual refresh with Cmd+R. Check rule syntax for errors that might prevent matching.

Import/Export Errors

Symptom: Error messages when importing libraries from other applications.

Solution: Ensure file format matches expected schema. Check for encoding issues in text files. Validate JSON syntax using external tools before importing.

Library Corruption

Symptom: Library showing incorrect data or crashing on launch.

Solution: Rebuild library database from Tools → Repair Library. Backup your library first. If issues persist, consider manual database rebuild by removing and re-adding game directories.
