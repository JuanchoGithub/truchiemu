Save State System

TruchiEmu's advanced save state system provides instant game saves and loads with visual thumbnails, metadata tracking, and cloud synchronization. Create checkpoints at any moment, organize saves into slot banks, and never lose progress with our automatic backup system.

Service Overview

The Save State system captures complete emulator state including RAM, CPU registers, GPU state, and audio buffers. Unlike traditional save states, TruchiEmu's implementation includes rich metadata, screenshot thumbnails, and integration with your game library for seamless organization and retrieval.

Save state capture and restore pipeline with thumbnail generation and backup

Core Features

  • Instant Captures: Save and load in less than 100ms with zero gameplay interruption
  • Visual Thumbnails: Automatic screenshot capture with customizable compression
  • Metadata Tracking: Track creation time, game position, and playtime between saves
  • Slot Banking: Organize saves into customizable banks for different playthroughs
  • Cloud Sync: Synchronize save states across devices via iCloud or custom providers
  • Backwards Compatibility: Import save states from other emulators with automatic conversion

Technical Architecture

The save state system is built on several core components:

  • Core Serializer: Captures complete emulator state from libretro cores
  • Compression Engine: LZ4-based compression for fast save/load operations
  • Thumbnail Generator: GPU-accelerated screenshot capture in real-time
  • Metadata Manager: Maintains save state database with rich indexing
  • Backup Scheduler: Automatic backup rotation with configurable retention policies
  • Cross-Platform Sync: Universal save state format compatible across macOS and iOS

Step-by-Step Configuration Guide

Basic Save State Usage

  1. Save State Creation

    Press Cmd+F1 (or your configured hotkey) to save to slot 1. The screen will flash briefly and a notification appears showing the saved thumbnail. You can configure up to 99 slots per bank.

  2. Load Save State

    Press F1 to load slot 1. Loading is instant - the game resumes exactly where saved. Use F2-F12 for slots 2-12, or use the save state browser for higher slot numbers.

  3. View Save State Browser

    Open the save state browser with Cmd+Shift+S to see all save states with thumbnails. Hover over states for metadata preview showing creation time, game location, and playtime.

  4. Organize with Slot Banks

    Create slot banks by pressing Cmd+B for Bank View. Organize saves by playthrough (e.g., Bank 1 for Speedrun, Bank 2 for 100% completion).

Advanced Save State Management

  1. Configure Rotate Mode

    In Settings → Save States → Rotate Mode, enable automatic slot rotation. Each new save automatically advances to the next slot, useful for quicksave checkpoints during difficult sections.

  2. Set Up Cloud Synchronization

    Enable iCloud sync under Settings → Save States → Cloud Sync. Choose sync frequency (immediate, hourly, daily) and which slot banks to synchronize. Configure conflict resolution preferences.

  3. Enable Automatic Backup

    Configure automatic backups in Settings → Save States → Backups. Set backup frequency (per save, hourly, daily) and retention count. Automatic backups prevent accidental overwriting.

  4. Configure Thumbnail Quality

    Adjust thumbnail compression and resolution in save state settings. Higher quality uses more disk space but provides better visual previews.

  5. Import External Save States

    Drag and drop save state files from other emulators into the save state browser. TruchiEmu automatically detects format (SRM, SV, etc.) and converts as needed. Manual mapping available for ambiguous conversions.

Save State Organization

  1. Name Save States

    Right-click any save state and select "Rename" to add descriptive labels. Use names like "Before Boss Fight", "Collected Secret Item", or "Speedrun Checkpoint" for easy identification.

  2. Create Save State Notes

    Add detailed notes to save states by double-clicking the state and entering text in the notes field. Document strategies, item locations, or completion goals.

  3. Auto-Cleanup Configuration

    Set up automatic cleanup in Settings → Save States → Cleanup to remove old states based on age or count. Configure rules to keep recent saves while removing outdated ones.

  4. Export Save States

    Select save states in browser and choose File → Export → Save States. Create shareable bundles for specific game sections or backup entire collections.

Configuration Examples

Example 1: Speedrunning Setup

// Save State Configuration
general {
    hotkey_save = "Cmd+F1"
    hotkey_load = "F1"
    show_notifications = true
    show_thumbnails = false         // Minimal HUD for speedruns
}

// Rotate Mode for Checkpoint Segments
rotate_mode {
    enabled = true
    slots_per_bank = 10
    auto_advance = true
    reset_on_bank_change = false
}

// Backup for Safety
backups {
    enable = true
    frequency = "per_save"          // Backup every single save
    retention_count = 5
    backup_location = "~/Documents/TruchiEmu/SaveBackups"
}

// Cloud Sync
cloud_sync {
    provider = "icloud"
    sync_on_save = true
    sync_on_load = false
    conflict_resolution = "prefer_newest"
}

// Performance
compression {
    algorithm = "lz4_fast"
    level = 1                       // Fastest compression
}

metadata {
    track_playtime = true
    track_location = true           // Document progress for practice
    add_timestamps = true
}

Troubleshooting

Save State Creation Fails

Symptom: Error message or crash when creating save state.

Solution: Check available disk space. Verify write permissions to save state directory. Try disabling compression in settings to isolate compression-related issues.

Save State Corruption

Symptom: Save state appears in browser but fails to load or crashes game.

Solution: Check for corrupted backup files in backup directory. Try importing save state into different emulator version. Verificationsum on save state files is missing or incorrect.

Cloud Sync Conflicts

Symptom: Error messages about conflicting save states during sync.

Solution: Review conflict resolution settings. Manually resolve conflicts in the sync dashboard. Consider using different slot banks for different devices to avoid conflicts.

Import/Export Problems

Symptom: Cannot import save states from other emulators.

Solution: Verify save state format is supported (SRM, SV, etc.). Check that ROM used matches the save state's ROM. Try manual conversion using tools → convert save state.

Thumbnail Generation Fails

Symptom: Save states appear without thumbnails in browser.

Solution: Check GPU drivers and Metal compatibility. Verify screenshot capture permissions in macOS Privacy settings. Try reducing thumbnail quality in settings.

Performance Impact

Symptom: Frame drops when creating save states frequently.

Solution: Enable asynchronous save state creation in advanced settings. Use faster compression algorithm (lz4 instead of zstd). Disable thumbnail generation for frequent saves.

Achievements Blocked

Symptom: Achievements not unlocking when loading save states.

Solution: You must be playing in Casual mode. Hardcore mode disables save states for achievement eligibility. Check your achievement settings to ensure save states are allowed.

Disk Space Issues

Symptom: Large collection of save states consuming too much disk space.

Solution: Enable automatic cleanup in settings. Enable stronger compression for existing saves. Consider reducing thumbnail quality or disabling screenshots for older states.
